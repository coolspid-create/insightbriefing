# [강의 자료] AI 기반 산업별 뉴스 브리핑 시스템 (Insight Briefing) 구축 보고서

본 문서는 'Insight Briefing' 프로젝트의 기획부터 설계, 구현 기술, 그리고 개발 과정에서 발생한 주요 문제 해결 과정을 상세히 기록하여 향후 교육 및 강의 자료로 활용할 수 있도록 구성되었습니다.

---

## 1. 프로젝트 개요 (Overview)

### 1.1 기획 배경
- **현상**: 정보 과잉 시대에 매일 쏟아지는 수만 건의 산업 뉴스 중 비즈니스에 핵심적인 인사이트를 추출하기 어려움.
- **해결책**: AI(LLM)를 활용하여 특정 섹터(IT, 반도체, 금융 등)의 핵심 뉴스만 선별, 요약하고 시각화된 대시보드와 자동화된 알림(텔레그램)을 제공하는 서비스.
- **목표**: 매일 오전 9시, 산업별 필수 뉴스 8개를 자동 생성하여 의사결정권자에게 전달.

### 1.2 주요 기능
- **다분야 뉴스 수집**: 섹터별 맞춤형 키워드를 통한 실시간 뉴스 검색 및 수집.
- **AI 인사이트 추출**: 단순 요약이 아닌 비즈니스 영향도(Impact)와 핵심 요약(Summary) 제공.
- **실시간 대시보드**: 최신 트렌드를 한눈에 파악할 수 있는 프리미엄 UI.
- **관리자 제어 시스템**: 검색어 설정, 금칙어 관리, 수동 업데이트 트리거 기능.
- **자동 알람**: 텔레그램 API를 통한 섹터별 채널 자동 발송.

## 2. 시스템 아키텍처 및 설계 의도 (Design Intentions)

### 2.1 기술 스택
- **Frontend**: React.js (Vite), CSS3 (Vanilla + Modern UI Patterns), Framer Motion (애니메이션).
- **Backend/Automation**: Node.js (Express), Node-schedule (스케줄러).
- **AI Engine**: OpenAI GPT-4o / GPT-4o-mini.
- **Database**: Supabase (PostgreSQL), JSON-based Local Storage (Dev/Caching).
- **Integration**: Telegram Bot API, Serper/Google Search API.

### 2.2 핵심 설계 의도
1.  **데이터 안정성 (Resilience)**: 외부 API(OpenAI, Search) 호출 실패 시 재시도 로직 및 로그 시스템 구축.
2.  **콘텐츠 품질 (Quality Control)**: 중복 제거(Deduplication) 로직과 24시간 이내 최신성 필터를 통해 정보의 가치 극대화.
3.  **UI/UX 심미성 (Aesthetics)**: 전문적인 금융/기술 리포트 느낌을 주기 위해 다크 모드 기반의 글래스모피즘(Glassmorphism) 및 세그먼트 애니메이션 적용.
4.  **확장성 (Scalability)**: 새로운 산업 섹터를 클릭 한 번으로 추가할 수 있도록 `config.json` 기반의 선언적 구조 채택.

## 3. 구축 기술 상세 (Implementation Details)

### 3.1 AI Pipeline Logic (automation/pipeline.js)
- **Step 1: Search**: 섹터별 지정된 쿼리(Query)로 뉴스 30개 이상 검색.
- **Step 2: Filter**: 중복 제목, 깨진 이미지 URL, 리디렉션 페이지 등을 정규식 및 유사도 검사로 제거.
- **Step 3: Score/Select**: AI에게 모든 뉴스 후보를 전달하여 관련성/영향도가 높은 상위 8개를 선정하도록 프롬프트 엔지니어링.
- **Step 4: Summary**: 선정된 뉴스를 바탕으로 제목(Title), 비즈니스 영향(Impact), 3줄 요약(Summary) 생성.

### 3.2 Frontend Design (frontend/src/)
- **Hero Section**: Framer Motion을 사용하여 실시간 인사이트 카드가 부드럽게 등장하는 애니메이션 구현.
- **Admin Dashboard**: 관리자가 실시간으로 DB 데이터를 수정하거나 파이프라인의 로그를 모니터링할 수 있는 인터페이스 제공.

## 4. 애로사항 및 트러블슈팅 (Challenges & Solutions)

### 4.1 OpenAI API 400 에러 및 JSON 형식 불일치
- **문제**: AI가 긴 기사 데이터를 처리하는 과정에서 설정된 토근 제한을 넘거나, 응답 형식이 유효하지 않은 JSON으로 돌아와 파싱 실패 발생.
- **해결**: `response_format: { type: "json_object" }`를 명시하고, 프롬프트 끝에 예시 데이터 구조를 강력하게 주입. 데이터 전송 전 문자열 이스케이프 처리를 강화함.

### 4.2 중복 뉴스 및 신선도 문제
- **문제**: 동일한 뉴스가 여러 언론사에서 발행될 경우 메인 페이지가 같은 내용으로 도배됨.
- **해결**: 제목 유사도 검사 로직(Levenshtein Distance 등)을 도입하고, 이미지 URL 해시를 비교하여 동일 기사 2차 필터링 수행. 섹터당 동일 기업 기사 최대 2건 제한 규칙 적용.

### 4.3 이미지 렌더링 및 깨진 링크
- **문제**: 수집된 뉴스 이미지 중 접근이 제한된 URL이 많아 대시보드 시각적 품질 저하.
- **해결**: 대체 이미지(Placeholder) 시스템 구축 및 뉴스 본문 내 og:image 태그 추출 로직 정교화.

### 4.4 데이터베이스 동기화 실패
- **문제**: 자동화 스크립트는 성공으로 표시되나 Supabase DB에는 반영되지 않는 고스트 업데이트 발생.
- **해결**: 비동기(Async/Await) 처리 과정에서 에러 핸들링을 세분화하여, DB Insert 실패 시 텔레그램으로 즉각 관리자 알림을 발송하도록 개선.

## 5. 강의 포인트 (Educational Takeaways)

1.  **AI 오케스트레이션**: 단순히 질문하고 답을 듣는 것이 아니라, 전처리-분석-후처리의 파이프라인을 구축하는 방법.
2.  **현대적 웹 프로그래밍**: 상태 관리보다는 데이터의 시각적 흐름(Data Flow)과 사용자 경험에 집중하는 UI 설계.
3.  **장애 대응 설계**: 모든 외부 API 호출은 실패할 수 있다는 가정하에 코드를 작성하는 방어적 프로그래밍(Defensive Programming) 실습.
4.  **B2B 서비스 UI**: 일반 유저보다 전문적인 데이터를 신뢰감 있게 전달하는 심미적 디자인 감각 고취.

## 6. 부록: 섹터별 상세 구성 (Appendix: Sector Details)

본 프로젝트는 다음과 같은 구체적인 섹터별 수집 로직을 가지고 있으며, 이는 AI 프롬프트에 직접 주입되어 필터링 성능을 극대화합니다.

### 6.1 신재생에너지 (태양광)
- **핵심 키워드**: 태양광, 신재생에너지, BIPV, BAPV, RPS, RE100.
- **선별 기준**: 국가 정책 변화, 대규모 투자, 안전사고/폐패널 이슈 등 실무 지식 중심.
- **제외 기준**: 단순 제품 출시 홍보, 주식 투자 분석, 부동산 관련 뉴스.

### 6.2 제품안전 (High Priority)
- **핵심 키워드**: KC인증, 리콜(Recall), 위해제품, 해외직구 안전관리.
- **선별 기준**: 국가기술표준원(KATS) 고시 개정, 리콜 통계, 어린이 제품 안전 기준 강화.
- **특이사항**: `isUrgent` 플래그를 통한 긴급 모니터링 대상.

### 6.3 인테리어·건설
- **핵심 키워드**: 제로에너지건축물(ZEB), 그린 리모델링, 자재 수급, 자재 성능 기준.
- **선별 기준**: 녹색건축법 개정, 창호/단열재 에너지 효율 등급제 변화.

### 6.4 AI·테크
- **핵심 키워드**: AI 윤리 가이드라인, SaaS 자동화(n8n), LLM Vertical 적용, 데이터 주권.
- **선별 기준**: AI 기본법 제정, 저작권 이슈, 기업용 AI 에이전트 도입 실태.

### 6.5 커머스
- **핵심 키워드**: 플랫폼법, 다크 패턴(눈속임 상술), 글로벌 플랫폼(알리/테무) 대응.
- **선별 기준**: 공정위 조사 동향, 이커머스 규제 법안, 소비자 보호법 개정.
